## [14:30] Motion-space FM jitter diagnosis + Consistency-FM attempt + GT-vel loss rescue
**Summary:** CFM v1 (motion-space + velocity/feature consistency) 降 jitter 部分有效（kick 227°→89°）但不够（sign-flip rate 60%，arms 64%）。根因：`dof_vel_cons` 只是 feature 自洽，CFM 只管 flow 直线不管 motion 平滑——都没直接约束"预测 motion 和 GT motion 的一阶/二阶差分一致"。停掉 CFM，启动 `g1_fm_velmatch_v1` 加 GT-matched vel+acc loss，只训 stage1 (80k)。

### What was done

- **实现 Consistency-FM**（`mld/train_g1_fm_cfm.py`, 430 行）：
  - 加 `CFMArgs.delta_t=0.02, cons_warmup_steps=20000`
  - Loss: v_rec + α·v_cons + w·f_cons + geometric
  - EMA target model 作 θ⁻
  - stages 80k+100k+100k
- **启动 g1_fm_cfm_v1** (GPU 1, PID 1014192)
- **50k checkpoint 诊断**（不等训完）：
  - max|joint| 改善：kick 227°→89°, run 172°→135°
  - max|vel| 改善：kick 1.81→0.53, run 1.19→0.70
  - 但 jump 肩膀 225°（v3 311°）还不够
  - 和 latent v1 比：stand/walk max|vel| 仍 2.4–2.7× 大
- **定量诊断 jitter**（npz 分析）：
  - sign-flip rate 52–64%，arms 最差 63.9%
  - 正常 motion 应 10–20%
  - 确认"单帧反复换方向"的高频振荡
- **根因定位**：
  - `dof_vel_cons` 只保证 feature self-consistency（dof_velocity channel == diff(dof_angle channel)）→ 两通道同步抖也是 0
  - CFM consistency 是 flow field 直线化，不是 motion 时序平滑

- **改 `mld/train_g1_fm.py`**：

  - `calc_loss` 新增 `vel_match_gt` + `acc_match_gt`（denormalize 到 raw radians 后算差分）
  - 加 7 个 monitor-only 指标（`mon/sign_flip_rate`, `mon/frame_delta_mean`, `mon/joint_abs_max` 等）
  - 重命名 `loss` → `total` 作为总 loss，保留 `loss` alias 兼容 train loop
  - 改 logging：`mon_*` 前缀的指标记到 `mon/` namespace
- **停 CFM v1**（kill 1014192）

  - `stage1_steps=80000, stage2_steps=0, stage3_steps=0` → 只训 stage 1 快速验证

### Problems & Solutions
- **Problem [13:30]:** CFM 50k 渲染显示 sign-flip rate 60%+，视觉上手臂明显抖动
  - **Solution:** 诊断现有 loss 发现 `dof_vel_cons` 是 feature self-consistency 不是 temporal smoothness。CFM 的 consistency 也只管 flow field。真正需要的是直接约束 pred_vel 匹配 GT_vel。改加 `vel_match_gt` + `acc_match_gt` loss。
  
- **Problem [13:20]:** `render_g1_rollout_fm.py` 无法加载 CFM checkpoint（yaml 含 `!dataclass:G1FMCFMArgs` tag）
  - **Solution:** 在 render 脚本加 `G1FMCFMArgs` fallback 导入 + 尝试两种 class 解析 yaml

  - **Solution:** 重命名 `loss` key → `total`（保留 `loss` alias 向后兼容）。加 `mon_*` 前缀分离监控指标。trainer 新增 sign_flip_rate / frame_delta / joint_abs_max 7 个 monitor。

### Key findings
- **`dof_vel_cons` loss 是伪约束**：`pred_dof_velocity_channel == diff(pred_dof_angle_channel)` 允许两个通道一起抖（feature 内部自洽 ≠ motion 平滑）。之前 v1/v2/v3 都用这个，实际没起平滑作用。
- **CFM 的 consistency 不等于 motion smoothness**：CFM 的 v_cons/f_cons 是让 flow trajectory 变直，跟单次预测内部的时序平滑无关。一个 1-shot x0 预测可以"沿直线 flow"但内部 8 帧依然抖。
- **Motion-space FM 真正需要的是 GT-matched 监督**：`||pred[t+1] - pred[t] - (gt[t+1] - gt[t])||` 这种差分匹配才强迫模型学到"frame-to-frame transition 该长什么样"。VAE latent 版不需要这个因为 decoder 隐式提供。
- 新指标 `mon/sign_flip_rate` 是诊断 jitter 最直接的数字，>40% = 抖，<20% = 平滑。

- **Paper 叙事线清晰**：FlowMotion 证明 x1-pred 方向对 → 我们 extend 到 K=1 compact motion + GT smoothness → ReFlow 进一步压到 K=1 质量。
