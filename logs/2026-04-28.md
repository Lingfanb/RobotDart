  - **Solution:** Split into 4 batches with explicit per-batch confirmation; user approved batches 1-3 then 4 separately. `mv` on same FS is metadata-only, fully reversible.

  - **Solution:** Fixed all with absolute paths via small Python script. The bug pre-dated the move (depth=4 was always wrong, should have been 5 for leaf-level files).

- **Problem [10:30]:** First cli.py NPZ smoke output: all segments of a clip got class 5 (stand)
  - **Solution:** Was passing clip-level `content_type_of_movement` to per-segment classifier → "standing" matched stand rule for ALL segments. Fixed: only use seg.text; fall back to clip type if seg.text empty.

- **Problem [10:35]:** wave_two_arms regex matched only 4/14 candidates
  - **Solution:** Pattern was `wave.*` but BONES descriptions say "waves" / "waving". Fixed: `\bwav(e|es|ed|ing)\b.*\b(two|both)\b.*hand`. wave_two_arms went from 4 → 36 matches.

- **Problem [11:34]:** BABEL VAD-A distribution heavily skewed positive (median +0.84, q75 +0.99)
  - **Solution:** Identified — per-action μ from BONES-only calibration is too low for BABEL real walks. Fix path documented (use _global μ/σ instead of per-action lookup, or recalibrate on BONES+BABEL combined). 2-pass design lets us iterate on regressor + re-run label_npz cheaply (~10 min vs 30 min full re-extract).

### Key findings

- **2-pass split is correct call** — VAD regressor still iterating; one full motion extract (30 min) followed by reusable cheap label passes (~10 min) gives ~3-5× faster iteration.
- **NPZ storage cost much lower than estimated**: spec said 27 GB, actual is 11 GB (np.savez_compressed gives ~60% compression on this data).
- **VAD-A bias on BABEL** is a real signal that per-action calibration needs cross-dataset thinking; BONES-only μ assumes BONES distribution.
- **Hot-reloadable YAML config** is paying off already — added `high_five` test class in 30 seconds without code change.
