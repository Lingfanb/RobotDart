### Problems & Solutions

- **Problem [11:20]:** 错误以为 "uniform t sampling" 是新做法 —— v6 args.yaml 显示 t_sampling 已是 logit_normal (是默认). 
  - **Solution:** 重新定位 v7 变量: v7 变的是 **logit_normal → uniform**, 这样变化才是真实 ablation row. v7 胜出 4/8 vs v6 1/8 → 确认 uniform 对 motion-space FM 比 logit-normal 好 (反直觉 vs SD3/Flux 结论).

- **Problem [14:00]:** First auto_eval 返回 exit code 2 ("retrain" decision), `set -e` 让 v6 eval 后 v7 训练没启动.
  - **Solution:** 改成 `|| echo "eval returned non-zero"` 让 eval decision 不 kill chain.

- **Problem [16:00]:** v10 extend stage2 30k→60k 反而变差 (110k: 1/8, 140k: 2/8).
  - **Solution:** 分析 rollout_prob = min(max_prob, (step - stage1)/stage2_steps). v10 同 step 时 stage2_steps=60k → rollout_prob 爬升更慢, 同样 110k step 时模型见到的 rollout 训练少于 v7. 确认 v7 的 30k stage2 是 sweet spot.

- **Problem [17:15]:** BONES-SEED 第一次下载只拿到 refs (12KB), HuggingFace 401/403 gated.
  - **Solution:** 用户申请 HF access 批准后重跑 `snapshot_download`. Patterns 一开始用 `g1/**` 但实际数据是 `g1.tar.gz` (单 tarball).

- **Problem [18:00]:** Fine-tune 测 vs from-scratch 测的 bias 问题.
  - **Solution:** 双阶段策略: (1) fine-tune 快速筛 (30k, 10 min/run), (2) 最终 paper table 用 from-scratch 110k 确认 winner. v7-scratch 现在跑中 (50-70 min).

### Key findings

**战略**
- **NMI target 锁定 + 硬 DDL 7/19** (13 周), 延期到 10/15 为 extension
- **VLM agent + tool-use 架构** 是 2026 趋势 core, ELLMER 已在 NMI 过稿, 我们有 precedent
- **HIAER 是直接对标** —— intro 必须明确 "我们是 robot motion VAD + handover task, 不是 library lookup"
- **Single GPU fine-tune fast iteration** → 多数情况可靠, 但架构/paradigm 改必须 from-scratch. Final paper 用 from-scratch.

**技术**
- uniform t sampling > logit-normal for motion-space FM (反直觉)
- 每个单独的 loss weight 调整都打破 v7 recipe → 留 4/8 做 "good enough" baseline
- BONES-SEED 是惊喜: 142k G1-ready motions, 35 万 segments, `content_uniform_style` 自带 neutral/injured/hurry/old 先验标签
- 但 BONES 92% neutral → 仍然需要 BEAT2 (76h, 8 情感) 补 affective diversity

**数据流**
- VAD 应该在 **时段级标注** (BABEL frame_ann, BONES temporal_labels), primitive 自动继承 overlap
- ABEE (~3200) 是唯一 GT VAD, 用作 kinematic VAD 回归器有监督 pretraining
- LLM API 成本: 35 万 segments dedupe 后约 30-60k → $5-15 on gpt-4o-mini
