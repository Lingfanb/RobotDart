# Plan · Goals & Roadmap

目标与执行路线图的统一入口。和其他目录的分工：

| 目录 | 干什么 |
|---|---|
| `plan/` | **我想去哪里** —— short/long-term goals, weekly plans, milestones |
| `notes/` | **怎么去** —— paper plan, architecture design, module build list |
| `knowledge/` | **已知的事** —— dataset/method/tool reference |
| `logs/` | **去了哪里** —— 每日工作日志 |
| `LOG_README.md` | TODO + 完成记录（active state） |

## 文件

| 文件 | 频率 | 内容 |
|---|---|---|
| [long_term.md](long_term.md) | 几月一更 | 1-3 年 vision，paper 提交后的下一步 |
| [short_term.md](short_term.md) | 每周一更 | 本周 focus + 下周 plan |
| [milestones.md](milestones.md) | 按需 | NMI 13 周里程碑表（时间维度） |
| [module_status.md](module_status.md) | 每周更 | 9 模块动态进度板（一行一个 M） |
| [risks.md](risks.md) | 按需 | 风险登记 + 应对 |

## 避免冗余 · 内容分工

不同文档虽然都讲"工作"，但角度不同：

| 问题 | 看哪里 |
|---|---|
| 某个 M 具体要做什么任务？ | [notes/module_build_list.md](../notes/module_build_list.md) |
| Paper 立意 / 4 个 contribution？ | [notes/paper_plan_nmi.md](../notes/paper_plan_nmi.md) |
| 某 M 的**定义和架构**？ | [notes/architecture_agent.md](../notes/architecture_agent.md) |
| 某 M **什么时候**做？ | [milestones.md](milestones.md) |
| 某 M **现在**在哪？ | [module_status.md](module_status.md) |
| 一次性完成的 analysis？ | [notes/analysis/](../notes/analysis/) |
| 完整 workflow 参考？ | [../knowledge/](../knowledge/) |
